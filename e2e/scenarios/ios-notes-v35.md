# Mobile E2E Сценарии: iOS Notes v35 — Мобильные взаимодействия

**Версия:** v35
**Дата:** 2026-04-01
**Платформа:** iOS (SwiftUI, iPhone 14, iOS 17+)
**Тип тестирования:** Mobile E2E — жесты, клавиатура, нативные UI-паттерны

> Эти сценарии проверяют МОБИЛЬНО-СПЕЦИФИЧНОЕ поведение приложения.
> API-сценарии и авторизация покрыты в `e2e/scenarios/mobile/ios-notes.md` (SC-M01, SC-M02, SC-01…SC-03).

---

## Предусловия (общие)

1. Симулятор запущен: `xcrun simctl boot "iPhone 14"`
2. Backend запущен на `http://localhost:4000` (`JWT_ENABLED=true`)
3. Приложение собрано и установлено на симулятор
4. Валидный токен записан в UserDefaults:
   ```bash
   TOKEN=$(curl -s -X POST http://localhost:4000/api/auth/dev-token | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")
   xcrun simctl spawn booted defaults write com.ultrawork.notes token "$TOKEN"
   ```

## SC-M01: Запуск приложения и отображение основного экрана

**Тип:** Mobile

**Предусловия:**
- Симулятор запущен, токен записан (см. общие предусловия)
- На backend присутствует хотя бы 1 заметка:
  ```bash
  curl -s -X POST http://localhost:4000/api/notes \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d '{"content":"Test note from E2E v35"}'
  ```

**Шаги:**
1. Запустить приложение на симуляторе (tap на иконку или через Xcode)
2. Дождаться завершения загрузки (исчезновения индикатора)
3. Проверить наличие на экране:
   - Заголовок навигации (Notes / Заметки)
   - Счётчик заметок (`notes_counter_text`)
   - Список заметок (`notes_list`) с хотя бы 1 элементом
   - Поле ввода новой заметки (`new_note_text_field`)
   - Кнопка добавления (`add_note_button`)
4. Сделать скриншот основного экрана

**Ожидаемый результат:**
Приложение запускается без крашей. Основной экран отображает навигационный заголовок, список заметок (загруженных с backend), счётчик и форму добавления. Индикатор загрузки (`isLoading`) присутствует в начале и исчезает после получения данных.

**Приоритет:** HIGH

**Режим:** manual

## SC-M02: Добавление заметки через UI и удаление свайпом

**Тип:** Mobile

**Предусловия:**
- Приложение запущено, список заметок отображается (SC-M01 пройден)
- Токен валиден

**Шаги:**
1. Запомнить текущее количество заметок (значение `notes_counter_text`)
2. Нажать на поле ввода `new_note_text_field` (tap)
3. Ввести текст: `"Test note from E2E v35"` через клавиатуру симулятора
4. Нажать кнопку `add_note_button` (tap на «+»)
5. Проверить, что новая заметка появилась в конце списка `notes_list`
6. Проверить, что счётчик увеличился на 1
7. Сделать скриншот списка с новой заметкой
8. Выполнить свайп вправо-влево (trailing swipe) по только что добавленной заметке
9. Нажать на красную кнопку «Delete» (или выполнить full swipe)
10. Проверить, что заметка исчезла из списка
11. Проверить, что счётчик уменьшился на 1
12. Сделать скриншот списка после удаления

**Ожидаемый результат:**
Заметка добавляется в список без перезагрузки экрана. Swipe-to-delete (`swipeActions, edge: .trailing, allowsFullSwipe: true`) открывает красную кнопку «Delete». После удаления элемент исчезает из `notes_list`, счётчик обновляется. Приложение не крашится.

**Приоритет:** HIGH

**Режим:** manual

## SC-M03: Скрытие клавиатуры при прокрутке списка

**Тип:** Mobile

**Предусловия:**
- Приложение запущено, в списке не менее 5 заметок:
  ```bash
  for i in 1 2 3 4 5; do
    curl -s -X POST http://localhost:4000/api/notes \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -d "{\"content\":\"E2E v35 заметка #$i\"}" > /dev/null
  done
  ```
- Список обновлён (pull-to-refresh или перезапуск)

**Шаги:**
1. Нажать на поле ввода `new_note_text_field` (tap) — клавиатура поднимается
2. Убедиться, что клавиатура отображается на экране
3. Начать прокрутку списка заметок вверх (scroll/swipe up на `notes_list`)
4. Наблюдать поведение клавиатуры во время прокрутки

**Ожидаемый результат:**
Клавиатура интерактивно скрывается по мере прокрутки списка (`scrollDismissesKeyboard(.interactively)`). Поле ввода теряет фокус. Список прокручивается плавно без артефактов. Текст, введённый до прокрутки, сохраняется в поле ввода.

**Приоритет:** MEDIUM

**Режим:** manual

## SC-M04: Поиск заметок через нативный SearchBar

**Тип:** Mobile

**Предусловия:**
- Приложение запущено, в списке присутствуют заметки с текстом `"Test note from E2E v35"` и другие заметки

**Шаги:**
1. Потянуть список вниз, чтобы раскрыть поисковую строку (или нажать на иконку поиска)
2. Ввести в строку поиска: `"v35"`
3. Проверить, что:
   - В `notes_list` отображаются только заметки, содержащие `"v35"`
   - `notes_counter_text` показывает формат «X из Y» (отфильтрованное / всего)
4. Сделать скриншот с результатами поиска
5. Очистить строку поиска (нажать «Cancel» или удалить текст)
6. Проверить, что список вернулся к полному отображению
7. Проверить, что счётчик показывает полное количество заметок

**Ожидаемый результат:**
Нативный `.searchable` фильтрует список в реальном времени без сетевых запросов. `NotesCounterView` отображает оба числа (filtered + total) при активном поиске. После сброса поиска список и счётчик возвращаются к исходному состоянию.

**Приоритет:** MEDIUM

**Режим:** manual

## SC-M05: Pull-to-refresh — перезагрузка списка с backend

**Тип:** Mobile

**Предусловия:**
- Приложение запущено, список заметок загружен (SC-M01 пройден)
- Токен валиден
- Через backend созданы заметки, которых нет в текущем UI (добавить напрямую):
  ```bash
  curl -s -X POST http://localhost:4000/api/notes \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d '{"content":"Test note from E2E v35 — refresh check"}'
  ```

**Шаги:**
1. Запомнить текущее количество заметок в списке (`notes_counter_text`)
2. Потянуть список `notes_list` вниз (swipe down от верхнего края) и удержать, пока не появится нативный индикатор обновления (spinner)
3. Отпустить список, дождаться завершения refresh (spinner исчезает)
4. Проверить, что новая заметка `"Test note from E2E v35 — refresh check"` появилась в списке
5. Проверить, что счётчик `notes_counter_text` увеличился на 1
6. Сделать скриншот списка после обновления

**Ожидаемый результат:**
Pull-to-refresh запускает `viewModel.fetchNotes()` через `APIService`, который выполняет `GET /api/notes` с токеном. Список обновляется данными с backend, новая заметка отображается в `notes_list`. Счётчик отражает актуальное количество. Поле `error_message` не отображается. Приложение не крашится.

**Приоритет:** HIGH

**Режим:** manual
